## ğŸš€ Quick Start

### Prerequisites

- **Docker** 20.10+ and **Docker Compose** 2.0+
- **Node.js** 20+ (for local frontend development)
- **Python** 3.11+ (for local backend development)
- **Git**

### Installation

#### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/visionwire-edtech.git
cd visionwire-edtech
```

#### 2. Configure Environment

```bash
# Copy environment template
cp .env.example .env

# Edit .env with your configuration
nano .env
```

**Required Configuration:**
- `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` for AI features
- `SECRET_KEY` for JWT authentication (generate with `openssl rand -hex 32`)
- Database credentials
- Email/SMS provider credentials (optional)

#### 3. Deploy with Docker (Recommended)

```bash
# Make deploy script executable
chmod +x scripts/deploy.sh

# Run deployment
./scripts/deploy.sh
```

**Or manually:**

```bash
# Start all services
docker-compose -f infrastructure/docker/docker-compose.yml up -d

# Run database migrations
docker-compose exec backend alembic upgrade head

# Seed initial data
docker-compose exec backend python scripts/seed-database.py
```

#### 4. Access the Platform

- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:8000
- **API Documentation**: http://localhost:8000/docs
- **Admin Panel**: http://localhost:3000/admin

**Demo Credentials:**
- Admin: `admin@visionwire.com` / `Admin@123`
- Teacher: `teacher@visionwire.com` / `Teacher@123`
- Student: `student@visionwire.com` / `Student@123`

### Backend Development

## ğŸ“Š API Documentation

Once the backend is running, visit:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### Key API Endpoints

**Authentication**
- `POST /api/v1/auth/register` - User registration
- `POST /api/v1/auth/login` - User login
- `GET /api/v1/auth/me` - Get current user

**Curriculum**
- `GET /api/v1/curriculum/boards` - List all boards
- `GET /api/v1/curriculum/subjects` - Get subjects
- `GET /api/v1/curriculum/tree` - Get curriculum tree
- `GET /api/v1/curriculum/learning-path` - Get personalized path

**Content Generation**
- `POST /api/v1/content/generate/notes` - Generate notes
- `POST /api/v1/content/generate/summary` - Generate summary
- `POST /api/v1/content/generate/package` - Generate full package

**Assessments**
- `POST /api/v1/assessments/create` - Create assessment
- `POST /api/v1/assessments/{id}/start` - Start assessment
- `POST /api/v1/assessments/attempt/{id}/submit` - Submit assessment

## ğŸ” Security Best Practices

1. **Change all default passwords** in production
2. **Use strong SECRET_KEY** (32+ characters)
3. **Enable HTTPS** with valid SSL certificates
4. **Configure CORS** properly for your domain
5. **Set up rate limiting** to prevent abuse
6. **Enable Sentry** for error tracking
7. **Regular database backups**
8. **Keep dependencies updated**
